# Contributing to Theourgia

Thank you for considering contributing.

Theourgia is in **planning phase** — there is not yet code to write against. The most valuable contributions right now are:

- **Reading and reacting to the plans.** [PROJECT_PLAN.md](PROJECT_PLAN.md), [ARCHITECTURE.md](ARCHITECTURE.md), and the per-phase plans in [plan/](plan/) describe the project's intended shape. Comments, corrections, and challenges are welcome via GitHub Issues.
- **Tradition-specific feedback.** If you practice a tradition the plan touches (Thelema, Greek theurgia, chaos magic, witchcraft, Hermetic ceremonial, folk traditions, Hindu, Buddhist, Egyptian reconstructionism, etc.) and see places where the plan misunderstands or oversimplifies your practice, please tell us — open an Issue.
- **Native-speaker review.** Theourgia is internationalized from day one. Native-speaker review of magickal terminology in non-English languages (Greek, Hebrew, Latin, Sanskrit, Arabic, Coptic, etc.) will be valuable as the corresponding modules land.

## Once code lands

When Phase 00 (Foundations) is complete, this file will be expanded with:

- Development environment setup
- Coding style guides (Python: Ruff + mypy; TypeScript: Biome + strict mode)
- Testing conventions
- Pull request workflow
- Review process
- ADR (Architecture Decision Record) process
- Plugin authoring guide (eventually)

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md). Theourgia welcomes magicians of all traditions; the CoC includes an explicit addendum about respect for divergent practice.
