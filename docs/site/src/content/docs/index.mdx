Community infrastructure for practicing magicians across traditions — Thelemites, chaos magicians, Greek theurgists, witches, Hermeticists, ceremonialists, folk practitioners, and adjacent paths.

It treats magickal practice as praxis worth recording rigorously, and treats data sovereignty as sacred.

<CardGrid>
  <Card title="For practitioners" icon="open-book">
    Calendars, astrology, divination, ritual logs, entities, sigils, gematria — every magickal artifact treated as portable, queryable data.
  </Card>
  <Card title="For self-hosters" icon="setting">
    Docker Compose deploy. Cloudflare R2 backups. One-click migrations. Zero telemetry, ever.
  </Card>
  <Card title="For communities" icon="puzzle">
    Federated networks. Group ritual coordinator. ActivityPub bridge to the wider Fediverse. Plugins from day one.
  </Card>
  <Card title="For developers" icon="pencil">
    AGPL-3.0 forever. Documented architecture. Stable plugin SDK. Tradition-respectful by design.
  </Card>
</CardGrid>

Theourgia is in **planning phase**. No runnable code yet. See [Status & roadmap](/start/status/) for the current progress through the 17-phase implementation plan.

The full implementation plan and feature catalog are public in the [GitHub repository](https://github.com/SAntonopoulou/theourgia).
