This section will hold developer documentation: contribution workflows, plugin authoring, internal APIs, testing patterns.

Theourgia is currently in **planning phase**. As code lands in Phase 01 onward, this guide will cover:

- Local development environment (devcontainer)
- Contribution workflow (branching, commits, PRs, reviews)
- Architecture deep-dives
- Plugin SDK (Python backend extensions + TypeScript frontend extensions)
- Federation protocol reference
- Testing patterns (unit / integration / E2E / property-based)
- ADR process (Architecture Decision Records)
- Release process (Conventional Commits → SemVer → tagged releases)

Until code lands, contribution is mostly about reading and reacting to the plans. See [CONTRIBUTING.md](https://github.com/SAntonopoulou/theourgia/blob/main/CONTRIBUTING.md) in the repository.
